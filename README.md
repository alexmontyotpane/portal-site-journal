# portal-site-journal

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于收集、整理和托管一系列独立的 HTML 页面。每个页面均为自包含的静态网页，不依赖特定后端服务或数据库。仓库本身不针对任何特定域名或网站，仅作为页面存档与展示的通用容器。

## 目录说明

```
portal-site-journal/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（图片、样式、脚本等）
├── index.html      # 可选的主入口页面（如页面索引或导航页）
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个独立页面，命名清晰以区分不同内容。
- **assets/**：按需组织，保持页面资源与页面文件分离。
- **index.html**：可放置简要的页面列表或跳转链接，方便浏览。

## 页面归档说明

归档的 HTML 页面遵循以下原则：

- 每个页面为完整独立 HTML 文件，包含必要的 CSS 和 JavaScript（内联或通过 assets 引用）。
- 页面内容不包含恶意代码、追踪脚本或广告。
- 页面可离线浏览，无需外部网络依赖（除非明确声明需要 CDN 等外部资源）。
- 归档目的为长期保存与展示，不承诺定期更新或内容审查。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面，或对已有页面进行改进。
- 提交前请确保页面无外部依赖或已明确标注依赖来源。
- 请勿提交包含个人隐私、敏感信息或违反法律法规的内容。
- 仓库维护者保留对提交内容进行审核、合并或拒绝的权利。

## 使用方式

1. 克隆仓库到本地：
   ```
   git clone https://github.com/your-username/portal-site-journal.git
   ```
2. 直接在浏览器中打开 `index.html` 或 `pages/` 下的任一 HTML 文件即可查看。
3. 也可通过 GitHub Pages 等服务将仓库部署为静态站点（需自行配置）。
